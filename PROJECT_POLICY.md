# Adept Project Policy

Adept is a team academic software project.

## Contribution rules

1. Do not push directly to `main` after branch protection is enabled.
2. Create a separate branch for every independently reviewable change.
3. Open a pull request before merging.
4. At least one other team member must review the pull request.
5. Required CI checks must pass before merging.
6. Never commit passwords, access tokens, private keys, `.env` files, or other secrets.
7. Code, tests, and relevant documentation must be completed before an issue is marked Done.

## Branch naming examples

- `feature/github-integration`
- `feature/dora-dashboard`
- `fix/refresh-token-rotation`
- `chore/update-ci`
- `docs/update-api-guide`
